## ADDED Requirements

### Requirement: 始業記録

システムは、ユーザーが当日の始業を記録できる手段を提供しなければならない（SHALL）。記録時、システムは現在日時を始業時刻として保存し、当日（`work_date`）のレコードが存在しなければ新規作成し、存在すれば更新しなければならない（MUST）。

#### Scenario: 当日の始業を初めて記録する

- **WHEN** ユーザーが当日まだ記録のない状態で始業を記録する
- **THEN** システムは当日の `work_report` を新規作成し、現在日時を `start_time` に保存し、ステータスを `start_recorded` にする

#### Scenario: 当日の始業を再記録する

- **WHEN** すでに当日の始業記録があり、ユーザーが再度始業を記録する
- **THEN** システムは同一日のレコードの `start_time` を新しい現在日時で更新する（重複レコードを作らない）

### Requirement: 終業記録と勤務時間計算

システムは、ユーザーが当日の終業を記録できる手段を提供しなければならない（SHALL）。記録時、システムは現在日時を終業時刻として保存し、当日の始業時刻が存在する場合は勤務時間を算出できなければならない（MUST）。

#### Scenario: 始業記録済みの日に終業を記録する

- **WHEN** 当日の `start_time` が記録済みの状態でユーザーが終業を記録する
- **THEN** システムは `end_time` に現在日時を保存し、`end_time - start_time` から勤務時間を算出し、ステータスを `end_recorded` にする

#### Scenario: 始業記録がない日に終業を記録する

- **WHEN** 当日の `start_time` が未記録の状態でユーザーが終業を記録する
- **THEN** システムは終業時刻を保存しつつ、勤務時間を未計算（不明）として扱い、ユーザーに始業未記録である旨を示す

### Requirement: 報告ステータス管理

システムは、各勤務日の報告状態を `not_started` / `start_recorded` / `start_mail_created` / `end_recorded` / `end_mail_created` のいずれかで表現しなければならない（MUST）。ステータスは記録およびメール作成イベントに応じて前進する。

#### Scenario: 始業メール作成でステータスが前進する

- **WHEN** ステータスが `start_recorded` の日に始業メールが作成（Outlook起動）される
- **THEN** システムは `start_mail_created_at` を記録し、ステータスを `start_mail_created` にする

#### Scenario: 終業メール作成でステータスが前進する

- **WHEN** ステータスが `end_recorded` の日に終業メールが作成される
- **THEN** システムは `end_mail_created_at` を記録し、ステータスを `end_mail_created` にする

### Requirement: 保存情報の範囲

システムは、勤務日・始業時刻・終業時刻・勤務区分・メモ・メール作成日時・終業メール本文（ヘッダ/フッタを除く）のみを保存対象とし、始業メール本文、終業メールのヘッダ/フッタ（テンプレートとして `mail_settings` 側に保持）、ならびに Microsoft 365 のパスワード、Outlook 認証情報、Graph アクセストークン、SMTP/アプリパスワードを保存してはならない（MUST NOT）。

#### Scenario: 記録保存時に認証情報を含めない

- **WHEN** ユーザーの始業・終業が記録される
- **THEN** 保存されるレコードには勤務日・時刻・勤務区分・メモ・メール作成日時・終業メール本文（ヘッダ/フッタ除く）のみが含まれ、いかなる外部メール認証情報も含まれない

#### Scenario: 始業メール本文は保存しない

- **WHEN** 始業メールが作成される
- **THEN** システムは始業メール本文を永続化せず、始業メール作成日時（`start_mail_created_at`）のみを記録する

### Requirement: 終業メール本文の保存

システムは、終業メール作成時に、ヘッダ・フッタを除いた本文（`end_body_template` の変数置換結果）を `work_reports` の `end_mail_body` に保存しなければならない（SHALL）。ヘッダ・フッタは `mail_settings` のテンプレートとして別途保持し、日次レコードには保存しない。

#### Scenario: 終業メール本文（ヘッダ/フッタ除く）を保存する

- **WHEN** 終業メールが作成される
- **THEN** システムは変数置換後の本文テンプレート部分を当日の `work_report` の `end_mail_body` に保存し、ヘッダ・フッタは保存しない

### Requirement: 勤務区分の記録

システムは、勤務区分を「オフィス」(`office`) または「リモート」(`remote`) の2択（固定）で記録できる手段を提供しなければならない（SHALL）。勤務区分はメールテンプレートの `{{work_style}}` に用いる。

#### Scenario: 勤務区分を選択して記録する

- **WHEN** ユーザーがその日の勤務区分にオフィスまたはリモートを選ぶ
- **THEN** システムは当日の `work_report` に `work_style` を保存し、メール生成時に `{{work_style}}` として展開する
